# 多实例部署指南 (Multi-Instance Deployment)

排班系统支持在同一台服务器/电脑上，运行多个完全独立的排班实例（例如不同的大科室各自拥有独立的系统、独立的数据和独立的访问端口）。

## 核心概念

通过配置以下环境变量，即可将系统隔离运行：
- `PORT`: 服务监听端口（例如 `3001`）。若设置为 `0`，系统将自动分配空闲端口。
- `DATA_DIR`: 数据存储目录。此目录用于存放该实例的 `data.json` 等业务数据。
- `INSTANCE_NAME`: 实例的唯一标识符（仅限英文、数字、下划线和连字符）。用于在后台服务、日志和开机自启快捷方式中区分不同实例。
- `RUNTIME_DIR` (可选): 运行时状态文件（`.server-pid`, `.server-port`, `server_launcher.log`）存放位置。默认会根据 `DATA_DIR` 或 `INSTANCE_NAME` 自动生成隔离的缓存目录。

## 快速上手：如何新建一个独立科室实例？

1. 在项目根目录下，找到 `start_server.bat` 文件。
2. 将其复制一份并重命名为您需要的名称（例如 `start_neike.bat`）。
3. 右键点击新复制的 `.bat` 文件，选择“编辑”（或使用记事本、VSCode等编辑器打开）。
4. 移除文件中配置变量前面的 `::` 注释符号，并修改为您需要的值，例如：
   ```bat
   set PORT=3001
   set INSTANCE_NAME=neike
   set DATA_DIR=data_neike
   ```
5. 保存并双击运行该 `.bat` 文件。您依然会看到熟悉的功能菜单（包含可见启动、静默后台、开机自启等），但此时所有的操作都会自动应用到您配置的这个独立实例上！

## 后台运行与开机自启

复制出来的独立启动脚本完全继承了主启动器的所有菜单功能。

如果您需要通过脚本实现“一键”动作（跳过菜单选择），可以通过附加数字参数调用您的独立脚本：
- `call start_neike.bat 1` : 可见窗口启动。
- `call start_neike.bat 2` : 静默后台启动。
- `call start_neike.bat 3` : 停止服务。
- `call start_neike.bat 4` : 查看运行状态。
- `call start_neike.bat 5` : 开启开机自启（静默模式）。
- `call start_neike.bat 6` : 关闭开机自启。

## 数据迁移与备份

- 由于每个实例的业务数据都在 `DATA_DIR` 中完全隔离，您只需打包压缩对应的 `DATA_DIR` 文件夹（例如 `data_neike/`）即可完成该科室的完整备份。
- 若要将旧系统的科室迁移到新实例，只需将旧的 `data.json` 放入新的 `DATA_DIR` 中即可。

## 注意事项

1. **端口冲突**：请确保不同实例配置了不同的 `PORT`（或直接使用 `0` 让系统自动分配）。
2. **实例名称**：请确保 `INSTANCE_NAME` 全局唯一，否则可能导致后台进程管理或开机自启快捷方式被相互覆盖。
3. **数据安全**：请定期备份各自的 `DATA_DIR`，避免因意外导致数据丢失。
